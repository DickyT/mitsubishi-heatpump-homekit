What it does

Turns CN105 into HomeKit

Control supported Mitsubishi heat pumps from Apple Home with local ESP32 firmware and a built-in web remote.

No cloud required

Local-first by design

Kiri Bridge talks to your indoor unit locally and exposes a WebUI at http://<device-ip>:8080/.

Recovery friendly

Browser flashing

If firmware recovery is needed, the web flasher writes Kiri packages over Web Serial from Chrome after metadata and checksum checks.

Setup flow

Power on

Plug in the Kiri Bridge. A rapid blue LED blink means it is ready for BLE WiFi provisioning.

Provision WiFi

Open BLE WiFi Provisioning, select the device, and send your 2.4 GHz WiFi credentials.

Open the portal

After provisioning, open the shown device portal at port 8080 and follow the HomeKit pairing instructions.

Connect CN105

Attach the included CN105 cable to the indoor unit and verify control from the WebUI and Apple Home.

Update when needed

Use the installer WebUI for normal package updates, or use the web flasher for a clean recovery installer.

Stay local

Once installed, Kiri Bridge keeps control on your LAN with HomeKit and the device WebUI.

Open source

Firmware you can inspect, fork, and repair.

Kiri Bridge follows an open firmware plus hardware kit model. The source code is public, releases provide ready-to-flash Kiri packages, and the hardware kit funds continued work.

Installer and recovery

Most buyers only need BLE provisioning. If you want to update after setup, use the installer portal. If the device is unreachable, use the browser flasher with the latest installer package from GitHub Releases.

Open BLE WiFi Provisioning Open Firmware Recovery

Tip: the web flasher does not automatically reboot the device after writing firmware. Restart it manually when the success dialog appears.
