Firmware recovery.

Re-flash a Kiri Bridge over USB. Use this if you bricked the device, want to roll back, or are loading the production firmware for the first time. Chrome or Edge required.

USB waiting Hardware not verified Package not selected
01

Pick the board.

Plug in the Kiri Bridge over USB. The browser reads the chip and MAC, then keeps the port reserved for the rest of the flow.

02

Board verified. Choose firmware.

The Kiri Bridge is reserved over USB. Choose a .kiri firmware package from GitHub Releases; the browser checks the manifest, sizes, and checksums before writing anything.

03

Write.

Confirm the board and package. When writing finishes, unplug the device manually — Chrome's USB stack does not recover well from a programmatic reset.
