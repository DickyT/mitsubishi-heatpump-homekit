# Support

Thanks for using Kiri Bridge.

## Best First Steps

- For normal setup, read [INSTALL.md](./INSTALL.md).
- For firmware recovery, read [FLASHING.md](./FLASHING.md).
- For firmware downloads, use GitHub Releases.
- For the hosted tools, use <https://kiri.dkt.moe>.

## Hardware Kit Support

Official support is focused on Kiri Bridge hardware kits. When asking for help,
please include:

- firmware version
- whether you are running installer or production firmware
- M5Stack ATOM Lite or other ESP32 hardware details
- Mitsubishi indoor unit model if known
- whether the LED is blinking blue, solid green, or doing something else
- what browser and operating system you used for provisioning or flashing
- logs or screenshots if available

For order interest, compatibility questions, or early hardware kit support,
email:

```text
hello@kiri.dkt.moe
```

## GitHub Issues

Use GitHub Issues for reproducible firmware, WebUI, provisioning, or recovery
problems. Please avoid posting WiFi passwords, HomeKit setup codes, or private
network details.

## Community Support Boundaries

Kiri Bridge is open source, so modified firmware and custom hardware are
welcome. However, custom builds and non-kit hardware may need community help
rather than direct hardware support.
